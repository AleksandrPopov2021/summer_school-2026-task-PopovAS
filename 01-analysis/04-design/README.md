# Дизайн данных — Скалодром «Вертикаль»

Проектирование модели данных клиентского мобильного приложения на основе анализа домена и требований (`01-analysis`).

## Содержание

| Документ | Описание |
|----------|----------|
| [er-model.md](./er-model.md) | ER-диаграмма, связи, перечисления, сводка доступа R/W |
| [entity-models.md](./entity-models.md) | Детальное описание сущностей с атрибутами и операциями приложения |
| [sequence-diagrams.md](./sequence-diagrams.md) | Sequence-диаграммы ключевых сценариев (запись, отмена, уведомления) |

## Ключевые выводы

- **Источник истины** — бэкенд-API; приложение не владеет расписанием, слотами, инструкторами и справочниками.
- **Приложение изменяет:** регистрацию клиента, создание/отмену записей, выбор проката, настройки уведомлений, оценки инструктора (post-MVP).
- **Приложение только читает:** слоты, зоны, инструкторов, прокатный фонд, статусы оплаты, допуски, причины отмены, параметры лояльности.
