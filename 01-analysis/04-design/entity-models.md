# Модели сущностей — Скалодром «Вертикаль»

> Детальное описание сущностей клиентского приложения с указанием операций чтения и записи.

**Обозначения:**
- 📖 **Только чтение (R)** — приложение получает данные из бэкенда, не изменяет
- ✏️ **Чтение и запись (R/W)** — приложение читает и инициирует изменения через API
- 📱 **Локально (L)** — хранится на устройстве, не является частью доменной модели бэкенда

---

## 1. Клиент (Client)

**Доступ:** ✏️ **R/W**  
**Описание:** Зарегистрированный пользователь мобильного приложения, записывающийся на тренировки.

| Атрибут | Тип | Обязательный | Доступ | Описание |
|---------|-----|--------------|--------|----------|
| `id` | UUID | да | R | Идентификатор в бэкенде |
| `full_name` | string | да | R/W | ФИО (BR-030) |
| `phone` | string | да | R/W | Телефон, ключ регистрации (BR-030) |
| `birth_date` | date | да | R/W | Дата рождения (BR-030) |
| `risk_consent_accepted` | boolean | да | R/W | Согласие на риск при первой записи (BR-031) |
| `completed_visits_count` | int | да | R | Количество посещённых тренировок |
| `is_loyal_client` | boolean | да | R | Бейдж «постоянный клиент» (BR-032) |
| `loyalty_discount` | decimal | нет | R | Скидка/бонус постоянного клиента |
| `late_cancellation_count` | int | да | R | Счётчик поздних отмен для санкций (BR-014) |
| `no_show_count` | int | да | R | Счётчик неявок (BR-015) |

**Операции приложения:**
| Операция | Направление | Use Case / FR |
|----------|-------------|---------------|
| Регистрация по телефону | W | UC-006, FR-026, US-017 |
| Подтверждение согласия на риск | W | FR-012, US-007 |
| Просмотр профиля и бейджа лояльности | R | FR-027, UC-007, US-018 |

**Связи:** 1 → N Запись, 1 → 1 Настройки уведомлений, 1 → N Допуск инструктора, 1 → N Оценка инструктора (post-MVP)

---

## 2. Слот тренировки (TrainingSlot)

**Доступ:** 📖 **Только чтение (R)**  
**Описание:** Единица расписания — конкретная тренировка, на которую записывается клиент.

| Атрибут | Тип | Обязательный | Описание |
|---------|-----|--------------|----------|
| `id` | UUID | да | Идентификатор слота |
| `starts_at` | datetime | да | Дата и время начала |
| `duration_minutes` | int | да | Длительность (~90 мин) |
| `capacity` | int | да | Вместимость группы (Y в «осталось X из Y») |
| `free_spots` | int | да | Свободные места (X) |
| `training_price` | decimal | да | Стоимость тренировки |
| `rental_tariff` | decimal | нет | Базовый прокатный тариф на слот |
| `slot_status` | enum | да | `active` \| `cancelled_by_gym` |
| `address` | string | да | Адрес скалодрома |

**Операции приложения:**
| Операция | Направление | Use Case / FR |
|----------|-------------|---------------|
| Получение расписания на 7 дней | R | UC-001, FR-001, FR-002, US-001 |
| Фильтрация по дате | R | FR-002 |
| Отображение отменённых слотов | R | FR-006, BR-019 |
| Проверка доступности записи (места, время) | R | FR-007, FR-008, US-023 |

**Связи:** N → 1 Зона, N → 1 Инструктор, N → 1 Скалодром, 1 → N Запись, 1 → N Прокатный фонд слота

**Бизнес-правила:** горизонт 7 дней (BR-027), запись не позднее 30 мин до начала (BR-006), при 0 мест — без кнопки записи (BR-008).

## 6. Запись / бронирование (Booking)

**Доступ:** ✏️ **R/W**  
**Описание:** Связь клиента и слота; центральная транзакционная сущность MVP.

| Атрибут | Тип | Обязательный | Доступ | Описание |
|---------|-----|--------------|--------|----------|
| `id` | UUID | да | R | Идентификатор записи |
| `client_id` | UUID | да | R | Ссылка на клиента |
| `slot_id` | UUID | да | R | Ссылка на слот |
| `booking_status` | enum | да | R/W* | Статус записи (*отмена клиентом — W) |
| `created_at` | datetime | да | R | Время создания |
| `cancelled_at` | datetime | нет | R | Время отмены |
| `uses_own_equipment` | boolean | да | R/W | Использует своё снаряжение |
| `cancellation_reason_id` | UUID | нет | R | Причина при отмене скалодромом |
| `rebooking_forbidden` | boolean | да | R | Запрет повторной записи на тот же слот (BR-018) |

\* Статусы `completed`, `no_show`, `cancelled_by_gym` устанавливает бэкенд; `cancelled_by_client` — по запросу приложения.

**Операции приложения:**
| Операция | Направление | Use Case / FR |
|----------|-------------|---------------|
| Создание записи | W | UC-002, FR-013, US-003–US-005 |
| Просмотр своих записей | R | FR-016, US-009 |
| Отмена клиентом | W | UC-003, FR-017–FR-019, US-010–US-011 |
| Обработка отказа бэкенда | R | FR-014 |
| Получение push при отмене скалодромом | R | UC-004, FR-020, US-012 |
| Предложение альтернативного слота | R | FR-022, US-014, US-024 |

**Связи:** N → 1 Клиент, N → 1 Слот, 1 → N Строка проката, 1 → 1 Оплата, 0..1 → Причина отмены

**Бизнес-правила:** атомарное бронирование на бэкенде (BR-003, BR-004), перенос не поддерживается (BR-013).

## 10. Оплата (PaymentInfo)

**Доступ:** 📖 **Только чтение (R)**  
**Описание:** Сумма и статус офлайн-оплаты по записи (value object, привязан к Booking).

| Атрибут | Тип | Обязательный | Описание |
|---------|-----|--------------|----------|
| `id` | UUID | да | Идентификатор |

| `training_amount` | decimal | да | Стоимость тренировки |
| `rental_amount` | decimal | да | Стоимость проката |
| `discount_amount` | decimal | нет | Скидка лояльности |
| `total_amount` | decimal | да | Итого к оплате |

## 15. Системные параметры (SystemConfig)

**Доступ:** 📖 **Только чтение (R)**  
**Описание:** Конфигурируемые параметры бизнес-логики (NFR-006).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `reminder_hours_before` | int | N часов до тренировки для push (BR-027) |
| `visits_for_loyalty` | int | N посещений для статуса постоянного клиента (BR-032) |
| `violations_for_sanctions` | int | N поздних отмен/неявок для санкций (BR-014) |
| `booking_cutoff_minutes` | int | Минимум минут до начала для записи (30, BR-006) |
| `cancellation_forbidden_minutes` | int | Запрет отмены менее чем за N минут (60, BR-012) |

**Операции приложения:** чтение для валидации UI и отображения правил.
