## Обзор

Логика создания записи на тренировку с экрана бронирования (SCR-005). Формирует тело запроса из черновика проката, проверяет согласие на риск и обрабатывает ответы API включая конфликты и бизнес-ошибки.

### User Story

> Как клиент, я хочу записаться на выбранную тренировку с нужным снаряжением,
> чтобы гарантировать себе место в группе.

### Бизнес-ценность

- Самостоятельная запись без администратора (BR-001, BR-003)
- Атомарное бронирование с защитой от двойной записи (BR-004)
- Актуализация UI при отказе бэкенда (FR-014)

## Описание логики

### Шаг 1: Проверка согласия на риск

Перед POST проверяется `client.risk_consent_accepted`. Если `false` — перенаправление на SCR-015 ([LOGIC-006](LOGIC-006_Подтверждение-согласия-на-риск.md)) без отправки запроса.

### Шаг 2: Формирование запроса

Тело `CreateBookingRequest` из `booking_draft`:
- `slot_id` — ID слота
- `uses_own_equipment` — флаг своего снаряжения (BR-002)
- `rental_lines` — массив `{ equipment_type_id, quantity }` из LOGIC-007

### Шаг 3: Успешное создание

При 201 — очистить `booking_draft`, показать подтверждение, переход на SCR-006. Push о подтверждении придёт асинхронно (FR-015, LOGIC-013).

### Шаг 4: Обработка 409 Conflict

Ответ `BookingConflictResponse` содержит актуальный `slot` (FR-014). Действия:
1. Обновить слот в `cached_slots`
2. Пересчитать LOGIC-004 на SCR-004
3. Показать `message` пользователю

### Шаг 5: Обработка 422

Коды: `NO_FREE_SPOTS`, `BOOKING_CUTOFF_EXCEEDED` — снек с `message`, обновление расписания.

### Шаг 6: Обработка 403

- `RISK_CONSENT_REQUIRED` → SCR-015
- `INSTRUCTOR_CLEARANCE_REQUIRED` → снек, обновление слота

## API запросы

### POST /bookings — `createBooking`

**Триггер:** Тап «Подтвердить запись» на SCR-005

**Headers:**

| Поле | Описание |
|------|----------|
| `Authorization` | `Bearer {access_token}` |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| `slot_id` | uuid | ID слота | `booking_draft.slot_id` |
| `uses_own_equipment` | boolean | Своё снаряжение | `booking_draft` |
| `rental_lines` | array | Позиции проката | LOGIC-007 |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Блокировка кнопки, spinner |
| Успех (201) | SCR-006, очистка черновика |
| Ошибка 403 | См. флоу по коду |
| Ошибка 409 | Обновить слот, снек, SCR-004 |
| Ошибка 422 | Снек `message`, обновить UI |
| Ошибка 5xx | Снек «Произошла ошибка. Попробуйте позже» |
| Ошибка сети | Снек «Нет соединения. Проверьте подключение к интернету» |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** валидный черновик и `risk_consent_accepted = true`, **Когда** пользователь подтверждает запись, **Тогда** отправляется POST /bookings |
| AC-002 | **Дано** успешный ответ 201, **Когда** запись создана, **Тогда** `booking_draft` очищен, открыт SCR-006 |
| AC-003 | **Дано** API возвращает 409, **Когда** конфликт обработан, **Тогда** UI слота обновлён из `slot` в ответе |
| AC-004 | **Дано** `risk_consent_accepted = false`, **Когда** пользователь нажимает «Подтвердить», **Тогда** открывается SCR-015 без POST |
| AC-005 | **Дано** 403 `RISK_CONSENT_REQUIRED`, **Когда** запрос отклонён, **Тогда** маршрут на SCR-015 |
| AC-006 | **Дано** 422 `BOOKING_CUTOFF_EXCEEDED`, **Когда** ошибка получена, **Тогда** снек с `message`, кнопка записи заблокирована |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| 409 Conflict | POST /bookings | Обновить слот, вернуть на SCR-004 |
| 403 RISK_CONSENT_REQUIRED | POST /bookings | SCR-015 |
| 422 NO_FREE_SPOTS | POST /bookings | Снек + обновление списка |
| 422 BOOKING_CUTOFF_EXCEEDED | POST /bookings | Снек + LOGIC-004 «Запись закрыта» |
