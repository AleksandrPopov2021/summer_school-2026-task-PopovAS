## Обзор

Логика изменения выбора проката в уже созданной активной записи (FR-028). Переиспользует форму LOGIC-007 и отправляет PATCH на бэкенд с проверкой доступности фонда.

### User Story

> Как клиент с активной записью, я хочу изменить выбор проката,
> чтобы взять или вернуть снаряжение при изменении планов.

### Бизнес-ценность

- Гибкость без отмены и повторной записи
- Актуальная стоимость в `PaymentInfo` после изменения
- Контроль остатков прокатного фонда на слот

## Описание логики

### Шаг 1: Условия доступности

Кнопка «Изменить прокат» видна только при:
- `booking_status = booked`
- До начала тренировки достаточно времени (рекомендация: `cancellation_policy.can_cancel` или отдельный флаг API)

### Шаг 2: Режим редактирования

SCR-005 открывается с параметрами `booking_id`, `slot_id` из записи. LOGIC-007 загружает актуальный фонд через [`getSlotRentalAvailability`](../api/openapi.yaml).

### Шаг 3: Отправка изменений

Тело `UpdateBookingRentalRequest`:
- `uses_own_equipment`
- `rental_lines`

Через [`updateBookingRental`](../api/openapi.yaml).

### Шаг 4: Обновление стоимости

Ответ `BookingDetail` содержит обновлённые `rental_lines` и `payment` — отобразить на SCR-007 (FR-011, FR-023).

### Шаг 5: Ошибка 422

`RENTAL_UNAVAILABLE` — показать `message`, обновить доступность, пользователь корректирует выбор.

## API запросы

### PATCH /bookings/{bookingId}/rental — `updateBookingRental`

**Триггер:** Тап «Сохранить изменения» в режиме edit_rental

**Headers:**

| Поле | Описание |
|------|----------|
| `Authorization` | `Bearer {access_token}` |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| `bookingId` | uuid | ID записи | SCR-007 |
| `uses_own_equipment` | boolean | Своё снаряжение | `booking_draft` |
| `rental_lines` | array | Позиции проката | LOGIC-007 |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Spinner |
| Успех (200) | SCR-007 с обновлёнными данными |
| Ошибка 422 `RENTAL_UNAVAILABLE` | Снек, обновить форму |
| Ошибка 403 | Снек «Изменение недоступно» |
| Ошибка 404 | Снек, SCR-006 |
| Ошибка 5xx | Снек «Произошла ошибка. Попробуйте позже» |
| Ошибка сети | Снек «Нет соединения. Проверьте подключение к интернету» |

### GET /slots/{slotId}/rental-availability — `getSlotRentalAvailability`

**Триггер:** Открытие формы редактирования

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Успех (200) | LOGIC-007: доступность чекбоксов |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** активная запись, **Когда** пользователь нажимает «Изменить прокат», **Тогда** открывается форма с текущим выбором |
| AC-002 | **Дано** пользователь изменил прокат, **Когда** PATCH возвращает 200, **Тогда** SCR-007 показывает новые `rental_lines` и `payment` |
| AC-003 | **Дано** позиция недоступна, **Когда** API возвращает 422 `RENTAL_UNAVAILABLE`, **Тогда** снек с `message` |
| AC-004 | **Дано** запись не активна, **Когда** SCR-007 отрисован, **Тогда** кнопка «Изменить прокат» скрыта |
| AC-005 | **Дано** успешное изменение, **Когда** пользователь на SCR-007, **Тогда** итоговая сумма обновлена |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| RENTAL_UNAVAILABLE | PATCH 422 | Обновить availability, остаться в форме |
| Запись отменена | 403 | SCR-006 |
| Конкурентное изменение фонда | 422 | Повторный выбор |
