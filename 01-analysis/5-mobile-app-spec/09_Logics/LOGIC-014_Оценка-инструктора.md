## Обзор

**Post-MVP.** Логика отправки оценки инструктора звёздами 1–5 на экране SCR-012 в течение 1–2 суток после завершения тренировки. Блокируется при отмене скалодромом и по истечении окна оценки.

### User Story

> Как клиент после тренировки, я хочу оценить инструктора,
> чтобы помочь другим выбрать качественное занятие.

### Бизнес-ценность

- Обратная связь для улучшения качества тренировок (BR-033)
- Отображение средней оценки до записи (BR-035, FR-004)
- Push-приглашение к оценке (FR-031)

## Описание логики

### Шаг 1: Условия доступности (BR-033)

Оценка доступна когда:
- `booking_status = completed`
- Прошло не более 1–2 суток после `slot.starts_at + duration` (проверка на бэкенде)
- Оценка ещё не отправлена

### Шаг 2: Блокировки (BR-034, FR-030)

- `booking_status = cancelled_by_gym` → UI «Оценка недоступна — тренировка отменена скалодромом»
- API 403 `RATING_NOT_ALLOWED_GYM_CANCELLED` — то же сообщение
- API 403 `RATING_WINDOW_EXPIRED` — «Срок оценки истёк»

### Шаг 3: Форма оценки

SCR-012: только звёзды 1–5, без текстового комментария (BR-033). Кнопка «Отправить» активна при `stars >= 1`.

### Шаг 4: Отправка

[`createInstructorRating`](../api/openapi.yaml) с телом `{ booking_id, stars }`.

### Шаг 5: Дубликат оценки

409 — «Оценка уже была отправлена», переход на SCR-006.

### Шаг 6: «Позже»

Закрытие SCR-012 без отправки. Push-приглашение может прийти повторно (если разрешено настройками LOGIC-011).

## API запросы

### POST /ratings — `createInstructorRating`

**Триггер:** Тап «Отправить оценку» на SCR-012

**Headers:**

| Поле | Описание |
|------|----------|
| `Authorization` | `Bearer {access_token}` |

**Параметры/Body:**

| Параметр | Тип | Описание | Значение/Источник |
|----------|-----|----------|-------------------|
| `booking_id` | uuid | Завершённая запись | Навигация / push |
| `stars` | integer | Оценка 1–5 | Выбор пользователя |

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Загрузка | Spinner на кнопке |
| Успех (201) | Снек «Спасибо за оценку!», SCR-006 |
| Ошибка 403 `RATING_NOT_ALLOWED_GYM_CANCELLED` | Сообщение о недоступности (BR-034) |
| Ошибка 403 `RATING_WINDOW_EXPIRED` | Сообщение об истечении срока |
| Ошибка 409 | Снек «Оценка уже была отправлена» |
| Ошибка 400 | Снек с `message` |
| Ошибка 5xx | Снек «Произошла ошибка. Попробуйте позже» |
| Ошибка сети | Снек «Нет соединения. Проверьте подключение к интернету» |

### GET /bookings/{bookingId} — `getBooking`

**Триггер:** Открытие SCR-012

**Обработка ответа:**

| Результат | Действие |
|-----------|----------|
| Успех (200) | Проверить `booking_status`, отобразить инфо о тренировке |
| Ошибка 404 | SCR-006, снек |

## Критерии приёмки

| ID | Критерий |
|----|----------|
| AC-001 | **Дано** завершённая запись в окне оценки, **Когда** пользователь выбирает 5 звёзд и отправляет, **Тогда** POST /ratings возвращает 201 |
| AC-002 | **Дано** запись отменена скалодромом, **Когда** открывается SCR-012, **Тогда** оценка заблокирована с сообщением BR-034 |
| AC-003 | **Дано** окно оценки истекло, **Когда** POST /ratings, **Тогда** 403 `RATING_WINDOW_EXPIRED` |
| AC-004 | **Дано** оценка уже отправлена, **Когда** повторный POST, **Тогда** 409 и снек |
| AC-005 | **Дано** push `rating_invitation`, **Когда** пользователь тапает, **Тогда** LOGIC-013 открывает SCR-012 |
| AC-006 | **Дано** пользователь нажал «Позже», **Когда** экран закрыт, **Тогда** возврат на SCR-006 без отправки |

## Обработка ошибок

| Тип ошибки | Контекст | Действие |
|------------|----------|----------|
| RATING_NOT_ALLOWED_GYM_CANCELLED | POST 403 | Блокирующее сообщение |
| RATING_WINDOW_EXPIRED | POST 403 / UI | Информационное сообщение |
| 409 Duplicate | POST | SCR-006 |
| stars не выбраны | Локально | Disable кнопки «Отправить» |
